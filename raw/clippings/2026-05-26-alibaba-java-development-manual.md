# 阿里巴巴 Java 开发手册（黄山版）

本手册是阿里巴巴集团技术团队的集体智慧结晶和经验总结，经历了多次大规模一线实战的检验及不断的完善，系统化地整理成册。现代软件行业的高速发展对开发者的综合素质要求越来越高，因为不仅是编程知识点，其它维度的知识点也会影响到软件的最终交付质量。

## 手册愿景

**码出高效，码出质量**

适当的规范和标准绝不是消灭代码内容的创造性、优雅性，而是限制过度个性化，以一种普遍认可的统一方式一起做事，提升协作效率。

## 六 大 维 度

| 维度 | 说明 | 约束力分类 |
|------|------|-----------|
| **编程规约** | 命名风格、常量定义、代码格式、OOP规范、集合处理、并发处理、控制语句、注释规约 | 强制 / 推荐 / 参考 |
| **异常日志** | 异常处理原则、日志级别规范 | 强制 / 推荐 / 参考 |
| **单元测试** | 测试质量保障 | 强制 / 推荐 / 参考 |
| **安全规约** | 鉴权、输入校验、敏感数据保护 | 强制 / 推荐 |
| **工程结构** | 应用分层、二方库依赖、服务器 | 强制 / 推荐 / 参考 |
| **MySQL数据库** | 建表规约、索引规约、SQL语句、ORM映射 | 强制 / 推荐 / 参考 |

## 约 束 力 分 级

- **强制**：禁止违反，违反后必须修改
- **推荐**：建议遵守，长期遵循有助于代码维护
- **参考**：了解即可，特殊场景可灵活处理

## 核 心 理 念

1. **代码质量是软件生命**：代码的字里行间流淌的是软件生命中的血液
2. **规范提升协作效率**：无规矩不成方圆，无规范不能协作
3. **减少踩坑**：质量的提升是尽可能少踩坑，杜绝踩重复的坑
4. **预防胜于治疗**：数据库表结构缺陷、工程结构混乱、安全漏洞应在开发阶段规避
