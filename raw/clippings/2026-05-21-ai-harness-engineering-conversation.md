# Spring Boot AI 驾驭工程模板讨论

本笔记记录了在 Spring Boot 项目中构建通用 AI Harness Engineering（AI 驾驭工程）模板的讨论。

## 背景
Harness Engineering（驾驭工程）在此语境下指的是当前使用 AI 编程时，规范 AI Agent 边界、代码规范、检查等方面的提示词工程（Prompt Engineering）。

## 核心设计：双维度约束

### 1. 代码规范驾驭 (Coding Harness)
通过设定严格的机器可读规则，约束 AI 在特定项目（如 Spring Boot）中安全、合规地输出高质量代码。
- **强制标记**：如要求必须使用 `// Generated By AI Start (Cursor)` 等标记。
- **技术栈与架构分层**：如规定 Controller 只能负责路由转换，Service 负责业务和事务，Mapper 复杂查询手写 XML。
- **不可变配置防御**：禁止修改敏感的 `pom.xml` 和 `application.yml` 配置。

### 2. 研发流程驾驭 (Process Harness)
通过工作流模板（思维链），强制 AI 写代码前进行思考设计，防止幻觉与过度工程。
- Step 1: 需求解析与依赖分析（关联现有功能、明确模糊点）
- Step 2: 存储与数据建模设计（DDL、数据量预估、实体映射）
- Step 3: API 契约与事务设计（RESTful 接口、分布式事务）
- Step 4: 编码实现与自测指导（按层实现，并给出测试关注点清单）
