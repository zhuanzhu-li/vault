# 纯文本知识库

由人类可读文本文件（通常为 Markdown）构成的个人知识系统，而非不透明数据库——为[[Digital Durability_数字耐久性|数字耐久性]]与[[User Data Ownership_用户数据所有权|用户数据所有权]]而优化。

## 为何用纯文本

- 任何编辑器、现在与未来都能阅读
- 便于 diff、grep 与脚本处理
- 经受任何单一阅读应用的[[Software Ephemerality_软件易逝性|软件易逝性]]

## 黑曜石：阅读器，而非牢笼

[[Steph Ango_斯蒂芬·安戈|斯蒂芬·安戈]]在[[Obsidian_黑曜石|黑曜石]]中写作，却把 `.md` 文件视为真正的产品。应用是透镜；库才是产物。这与[[File Over App_文件优于应用|文件优于应用]]一致。

## 本库结构

- `raw/` — 精选来源（对 agent 只读）
- `wiki/` — 用 Markdown 与[[wikilinks]]合成的概念与实体
