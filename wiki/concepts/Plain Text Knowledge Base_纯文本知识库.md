# 纯文本知识库

由人类可读文本文件（通常为 Markdown）构成的个人知识系统，而非不透明数据库——为[数字耐久性](Digital%20Durability_数字耐久性.md)与[用户数据所有权](User%20Data%20Ownership_用户数据所有权.md)而优化。

## 为何用纯文本

- 任何编辑器、现在与未来都能阅读
- 便于 diff、grep 与脚本处理
- 经受任何单一阅读应用的[软件易逝性](Software%20Ephemerality_软件易逝性.md)

## 黑曜石：阅读器，而非牢笼

[斯蒂芬·安戈](Steph%20Ango_斯蒂芬·安戈.md)在[黑曜石](Obsidian_黑曜石.md)中写作，却把 `.md` 文件视为真正的产品。应用是透镜；库才是产物。这与[文件优于应用](File%20Over%20App_文件优于应用.md)一致。

## 本库结构

- `raw/` — 精选来源（对 agent 只读）
- `wiki/` — 用 Markdown 与[wikilinks](wikilinks)合成的概念与实体
