# 内核网络参数

Kubernetes 集群中必须正确配置的一组 Linux 内核网络参数，否则 [kube-proxy iptables 模式](kube-proxy%20iptables%20模式_kube-proxy%20iptables模式.md) 的规则无法生效，Pod 互通和 Service 访问将被阻断。

## 关键参数

所有节点均需配置 `/etc/sysctl.d/k8s.conf`：

```ini
# 让网桥流量经过 iptables（CNI 网桥/overlay 必需）
net.bridge.bridge-nf-call-iptables = 1
net.bridge.bridge-nf-call-ip6tables = 1

# 开启 IP 转发（Pod 跨节点、Service 访问必需）
net.ipv4.ip_forward = 1
```

生效命令：`sysctl --system`

## 参数详解

### bridge-nf-call-iptables

启用后，Linux 网桥上的二层流量会被注入 iptables 处理。CNI 插件（如 [Flannel](Flannel_Flannel.md) 的 VXLAN、[Calico](Calico_Calico.md) 的网桥模式）依赖此参数使 iptables 规则覆盖容器网络流量。

### ip_forward

开启后允许节点转发数据包。Pod 跨节点通信和 Service ClusterIP 访问依赖 IP 转发能力。
