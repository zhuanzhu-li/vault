# 造工具

## 定义

"造工具"不是非得从零造一个基础模型，也不是非得开一家硅谷公司。而是你开始能够把重复任务抽象成模板，把高频动作封装成流程，把经验变成结构，把结构变成系统，把系统变成可复用的工具链。

## 造工具 vs 用工具 vs 组合工具

| 层级 | 特征 | 能力要求 |
|------|------|---------|
| **用工具** | 只会调用现成工具 | 最低 |
| **组合工具** | 能把多个工具串联使用 | 中等 |
| **造工具** | 能自己封装流程、构建Agent、设计pipeline | 最高 |

## 造工具的进阶路径

```
用工具 → 组合工具 → 创造工具
```

## 造工具的具体表现

| 场景 | 造工具前的状态 | 造工具后的状态 |
|------|--------------|---------------|
| 写文章 | 每次从零开始 | 有内容模板可复用 |
| 做研究 | 手动复制粘贴 | 有 data pipeline + prompt pipeline |
| 做交易 | 凭情绪临场判断 | 有观察清单、因子框架和执行路径 |
| 做产品 | 一段段散乱试错 | 把反馈、代码、页面、支付、数据流串成闭环 |

## 造工具的本质

> 把人的经验固化为机器可以重复执行的结构。

这也是工业革命真正发生时的底层逻辑。斧子、锄头、流水线、数据库、互联网基础设施，本质上都不是"灵光一闪"的浪漫结果，而是大量重复劳动之后，对规律的提炼、固化与再部署。

## AI对造工具的加速

| 传统方式 | AI时代 |
|---------|--------|
| 半年到一年沉淀一个经验 | 几天到几周快速试错形成闭环 |
| 试错成本高 | 试错成本大幅降低 |

## 造工具的循环

```
试错 → 修正 → 固化 → 复用 → 扩展
```

AI 最强的地方，不是代替你做一次事，而是让你更快形成这个循环。

## 与[AI时代生产力公式](AI时代生产力公式.md)的关系

"造工具"对应公式中的**第四个层次**：

```
生产力 = (思维 × 方法 × 逻辑 × 执行) × (工具 × 技巧 × 组合 × 造工具)
                                    └── 这是最高层次 ──┘
```
