## 一、性能调优与线上故障排查 ⭐⭐⭐

### 目标
能够独立定位和解决生产环境的性能问题。

### 知识清单

| 专题 | 细分 | 优先级 |
|------|------|--------|
| **JVM 调优** | GC 日志分析、MAT/OOM排查、G1 vs ZGC 选择、JVM 参数调优 | ⭐⭐⭐ |
| **CPU 高负载排查** | Arthas/async-profiler、火焰图、热点代码定位 | ⭐⭐⭐ |
| **内存泄漏排查** | Heap Dump、MAT 分析、集合类泄漏、静态引用 | ⭐⭐⭐ |
| **接口响应迟缓** | 链路追踪（Skywalking/Zipkin）、慢 SQL 定位 | ⭐⭐⭐ |
| **数据库慢查询** | Explain 执行计划、索引优化、连接池调优 | ⭐⭐⭐ |
| **线程阻塞分析** | jstack、线程状态分析、锁竞争定位 | ⭐⭐ |

### 参考已有
- [java-interview-jvm](Java开发面试/java-interview-jvm.md)

### 实践建议
- 使用 ` Arthas ` 对本地 Demo 进行模拟排查
- 对标阿里 Java 应用性能手册的调优标准

## 二、系统设计与架构能力 ⭐⭐⭐

### 目标
能够设计中高复杂度的系统方案，回答 "如何设计 XXX" 类问题。

### 知识清单

| 专题 | 核心考点 | 优先级 |
|------|----------|--------|
| **系统设计基础** | 高可用、可扩展、易维护性、CAP/BASE 理论 | ⭐⭐⭐ |
| **分布式 ID 生成** | Snowflake、UUID、Leaf、C Segment | ⭐⭐⭐ |
| **分布式锁** | Redis 锁、ZooKeeper 锁、红锁、StampedLock | ⭐⭐⭐ |
| **分布式事务** | 2PC/3PC、TCC、Saga、本地消息表、Seata | ⭐⭐⭐ |
| **一致性哈希** | Hash 环、虚拟节点、Memcached/Redis Cluster | ⭐⭐ |
| **读写分离 + 分库分表** | ShardingSphere、MyCAT、Sharding-JDBC | ⭐⭐ |
| **接口幂等设计** | Token + Redis、唯一索引、乐观锁版本号 | ⭐⭐⭐ |
| **延迟任务队列** | 时间轮、Redis ZSet、RabbitMQ 延迟消息 | ⭐⭐ |
| **秒杀系统设计** | 库存预扣、限流、MQ 削峰、Redis 原子操作 | ⭐⭐ |

### 推荐学习路径
1. 阅读《系统设计面试》或《Designing Data-Intensive Applications》
2. 对每个主题尝试画出架构图
3. 参考公司内部或开源项目的实现方案

## 三、框架与中间件深入 ⭐⭐⭐

### 目标
不仅会用，还要理解底层原理，能进行二次定制。

| 框架/中间件 | 需要深入理解的点 | 优先级 |
|------------|----------------|--------|
| **Spring** | IoC/AOP 原理、Bean 生命周期、循环依赖解决、事务原理 | ⭐⭐⭐ |
| **Spring Boot** | 自动配置原理、-starter 机制、事件监听机制 | ⭐⭐ |
| **MyBatis** | 插件机制（二级缓存/分页）、SQL 执行流程 | ⭐⭐ |
| **Dubbo** | 服务发现、负载均衡、序列化、RPC 原理 | ⭐⭐⭐ |
| **RocketMQ / Kafka** | 消息顺序、事务消息、幂等消费、重试机制 | ⭐⭐⭐ |
| **Redis** | 数据结构底层实现、集群脑裂、缓存穿透/击穿/雪崩 | ⭐⭐⭐ |
| **MySQL** | InnoDB 锁机制、MVCC、索引底层（B+树）、主从复制原理 | ⭐⭐⭐ |

### 参考已有
- [java-interview-frameworks](Java开发面试/java-interview-frameworks.md)

| **Kubernetes** | Pod/Service/Deployment、Helm、健康检查、资源限制 |
| **CI/CD** | Jenkins、GitLab CI、ArgoCD、蓝绿部署/金丝雀发布 |

## 七、学习路线图（建议周期：8-12 周）

| 周次 | 重点 | 交付物 |
|------|------|--------|
| 第 1-2 周 | 性能调优（JVM/GC/Arthas） | 完成 3 个模拟故障排查报告 |
